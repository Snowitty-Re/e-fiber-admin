# ADR-0002: 多币种定价采用按币种独立存价

- 状态：Accepted
- 日期：2026-06-18
- 关联：`docs/02` §2.2、`docs/03` §5.4、`docs/05` §模块1

## 背景

外贸多市场定价是刚需：不同币种/市场需精准差异化定价，而非汇率换算。B2B 报价场景对展示价稳定性要求高。

## 决策

价格按币种独立存储：`variant_price(variant_id, currency_code, amount)`，每个启用币种一条记录，`(variant_id, currency_code)` 唯一。金额以最小货币单位整数存储。

## 备选

| 方案 | 否决理由 |
|---|---|
| 基础币种 + 实时汇率换算 | 展示价随汇率波动、不精准，B2B 报价不可用 |
| 基础价 + 币种覆盖表（混合） | 复杂度适中，但默认仍依赖换算；MVP 不需要，未来可作升级 |

## 后果

- 正面：价格精准、可设市场差异化价、展示稳定。
- 负面：写入与存储略重（每币种一行）；新增币种需补价。
- 升级位：未来可叠加 `price_list` 覆盖层（本期不做，`docs/02` §3 明确非目标）。
